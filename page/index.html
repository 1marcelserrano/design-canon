design-canon · uma skill do Claude

Tokens com disciplina.

Você tem uma paleta e umas fontes.
Eu devolvo tokens fiéis. Seis formatos, sem inventar nada.

TL;DR
  • Pedir tokens a um modelo dá saída infiel: cor renomeada, rampa de cinza inventada, formato errado, e diferente a cada vez.
  • A design-canon separa disciplina (o método) de marca (os valores) e roda um gerador determinístico: mesma entrada, mesma saída.
  • Três disciplinas (Vignelli, suíça, blank): seis formatos de token, zero rede e zero credencial.

Design tokens são a ponte entre a marca e o código. A design-canon atravessa essa ponte sem chutar: os valores são os que você declara; o método é o preset que você escolhe.

Camada 1 · O problema

▾

Quem gera token costuma designar demais

Pede pra um modelo "gerar os design tokens da minha marca" e ele entrega algo bonito e infiel. Renomeia suas cores (primary vira lime), inventa uma rampa de cinza que não existe no seu DS, cria tints e shades que você nunca aprovou.

A causa é simples: gerar tokens mistura duas decisões que deviam ser separadas. Qual o método (quantos tamanhos de tipo, que grids, como a cor se comporta) e quais os valores (seus hexes, suas fontes). Quando viram uma coisa só, o modelo preenche os buracos chutando.

O atrito

E o chute é justo a parte que você não controla. Para um handoff de dev, isso é ruído: você queria os seus valores, normalizados, num formato fixo. Não uma reinterpretação nova a cada rodada.

Camada 2 · O mecanismo

▾

Disciplina ⊥ marca

O modelo central é uma ortogonalidade. A disciplina é o método: razão da escala de tipo, set de grids, pesos de régua, semântica de cor, fallback de fonte. É a régua. A marca são os valores: paleta, fontes, tamanho-base. É o conteúdo.

Você renderiza os valores da sua marca através de uma disciplina escolhida. A disciplina nunca troca seus valores: ela impõe a estrutura e avisa, fora do arquivo, quando a marca viola o método. Vignelli, por exemplo, virou um preset: o rigor do Canon aplicado a qualquer paleta, inclusive a sua.

O que vem na caixa

0disciplinas
(vignelli · swiss · blank)
0formatos de token
(css · scss · tailwind · json · w3c · style-dictionary)
0dependências
(sem rede, sem credencial)

Números do gerador, não de marketing: 3 presets embutidos, 6 emissores, zero rede. Verificável em --list-presets.

Camada 3 · Como funciona

▾

Da marca ao token, em um comando

O gerador é um script Python sem dependência. Cinco passos, determinísticos:

01

Descreve a marca

Paleta, fontes e tamanho-base num JSON, ou direto por flags (--primary, --base, --face).

02

Escolhe a disciplina

vignelli (o Canon), swiss (estilo internacional) ou blank (passthrough sem opiniões).

03

O gerador funde

Estrutura do preset + valores da marca. Precedência clara: flag > config > default do preset.

04

Avisa, fora do arquivo

Se a marca dilui a cor-identificador ou estoura o limite de tamanhos, o aviso sai em stderr, nunca no token.

05

Exporta o formato

CSS, SCSS, Tailwind v4 @theme, JSON, W3C Design Tokens ou Style Dictionary. O mesmo DS, o formato que seu time usa.

Camada 4 · Um exemplo

▾

Um exemplo real

Você descreve a marca num JSON e pede o formato. A mesma marca pode sair pela disciplina blank (sem opiniões) ou vignelli (dois tamanhos, régua, grid). Aqui, o DS da casa em Tailwind v4:

Entrada · ds.json

{ "palette": { "primary": "#B4C636", "bg": "#0A0A0E" }, "fonts": { "sans": "Inter Tight" }, "base_px": 16 }

Saída · design_canon.py --config ds.json --format tailwind

@theme {
  --color-primary: #B4C636;
  --color-bg: #0A0A0E;
  --font-sans: 'Inter Tight', 'Liberation Sans', Arial, sans-serif;
  --text-base: 16.0px;
  /* … */
}

primary continua primary. Nenhuma cor renomeada, nenhuma rampa inventada. O fallback 'Liberation Sans' entra sozinho, pra fonte não quebrar em render headless.

Pra fechar

▾

Vignelli virou um preset. O método vale pra qualquer marca.

Pega a disciplina que você respeita e aplica na marca que você tem. O token sai fiel, no formato que seu time usa. Sem rede, sem credencial, sem chute.
